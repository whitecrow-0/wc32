wc32

Text Encoder & Decoder — Python library with optional salt support.

Install

pip install wc32

Usage

Basic

from wc32 import encode, decode

encoded = encode("Hello, World!", "my_secret_key")
decoded = decode(encoded, "my_secret_key")
print(decoded)  # Hello, World!

Non-deterministic (recommended)

encoded = encode("Hello", "key", salt=True)
# Each call produces different output
decoded = decode(encoded, "key")  # Hello

Custom salt

encoded = encode("Hello", "key", salt="deadbeef12345678")
decoded = decode(encoded, "key")  # Hello

What is salt?

A salt is a random value mixed into the encoding process before your key is applied. It has nothing to do with making output "more secret" — its sole job is to make the same input produce a different output every time.

Without salt — deterministic

Encoding "hello" with key "mykey" will always produce the exact same string. If someone sees two identical encoded values they immediately know the original inputs were identical too. That leaks information — even without breaking the encoding. 

encode("hello", "mykey")  # → always "Th.T7.T-.T*.TD"
encode("hello", "mykey")  # → always "Th.T7.T-.T*.TD"  ← identical, information leaked

With salt — non-deterministic

A random 8-character hex prefix is generated and folded into the keystream before encoding. Every call produces a completely different string even for identical inputs. The salt is stored at the start of the encoded output so decode() can recover it automatically — you never have to pass the salt separately. 

encode("hello", "mykey", salt=True)  # → "a3f9c1b2.Xk.mR.pQ.nT.wZ"
encode("hello", "mykey", salt=True)  # → "e7d20a5f.Yj.oS.qL.vU.xN"  ← different every time

decode("a3f9c1b2.Xk.mR.pQ.nT.wZ", "mykey")  # → "hello"  ✓
decode("e7d20a5f.Yj.oS.qL.vU.xN", "mykey")  # → "hello"  ✓

When should you use it?

Use salt=True whenever the same value might be encoded more than once and you don't want observers to correlate the outputs — e.g. storing tokens, session identifiers, or any repeated string. Skip it only when you intentionally need reproducible output (caching, deduplication).

Custom salt

You can pass your own 8–32 character hex string as the salt. This is useful for deterministic but key-varied encoding — two parties can agree on a salt in advance and both produce identical output. 

encode("hello", "mykey", salt="deadbeef")
decode("deadbeef.Xk.mR.pQ.nT.wZ", "mykey")  # → "hello"
Remember — salt prevents output correlation, it does not strengthen the encoding itself. wc32 is obfuscation, not encryption. For secrets that must be kept, use a proper encryption library such as cryptography.
